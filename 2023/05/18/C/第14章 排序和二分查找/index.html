第十四章：排序和二分查找

排序的前提是”比较”，排序的目的往往是”查找”。这一章，我们主要给大家介绍一些经典的基于比较的排序算法，以及经常被人低估的二分查找算法。

1. 排序

1.1 如何评估一个排序算法

我们可以从三个维度去分析一个排序算法：时间复杂度、空间复杂度和稳定性。

  1. 时间复杂度
  • 最好情况
  • 最坏情况
  • 平均情况
  • 系数和低阶项
  1. 空间复杂度

我们需要重点关注原地排序(store in place)，也就是空间复杂度为 O(1) 的排序。

  1. 稳定性

数据集中”相等”的元素，如果排序前和排序后的相对次序不变，那么这个排序算法就是稳定的。稳定性是排序算法一个很重要的指标。

我们通过一个例子来说明这个问题：在电商系统中，经常需要查看订单数据，以做数据统计。现在有这样一个需求，先对订单按金额从大到小排序，如果金额相同，再按下单时间从大到小排序。

解决思路：先按下单时间从大到小排序，再用稳定的排序算法，按金额从大到小排序。

1.2 选择排序

选择排序(Selection sort)是一种简单直观的排序算法。

其基本思想是：首先在未排序的数列中找到最小(或最大)元素，然后将其存放到数列的起始位置；接着，再从剩余未排序的元素中继续寻找最小(或最大)元素，然后放到已排序序列的末尾。以此类推，直到所有元素均排序完毕。

1.3 冒泡排序

冒泡排序(Bubble sort)也是一种简单直观的排序算法。

其基本思想是:从左到右，依次比较相邻的两个元素，如果它们逆序，则交换这两个元素。每比较一轮(冒泡一次)，最大的元素就会”沉”到数列的末尾，而较小的元素会慢慢”浮”到数列的前面。因此,经过 N-1 (假设有N个元素)次冒泡，数组就排好序。

1.4 插入排序

插入排序(Insertion sort)也是一种简单直观的排序算法。

其基本思想是：构建有序序列。对于未排序数据，在已排序序列中从后向前扫描，找到相应位置并插入。

插入排序类似于摸牌并将其从小到大排列。每次摸到一张牌后，根据其点数插入到确切位置。

分析

  1. 时间复杂度

    a. 最好情况：原数组有序。O(n)

    ​ 比较次数：n-1

    ​ 交换次数：0

    b. 最坏情况：原数组逆序。O(n2)

    ​ 比较次数：1 + 2 + … + (n-1) = n(n-1)/2

    ​ 交换次数：1 + 2 + … + (n-1) = n(n-1)/2

    c. 平均情况：O(n2)

    ​ 直接分析十分复杂：n 个元素有 n! 种排列，然后分别统计每种排列比较和交换的次数，再计算平均值。

​ 这里我们采用逆序对的方式来分析，首先来看下相关的定义：

​ 有序元素对：arr[i] <= arr[j], i < j

​ 逆序元素对：arr[i] > arr[j], i < j

​ 有序度：数列中有序元素对的个数

​ 逆序度：数列中逆序元素对的个数

​ 满有序度：数列排好序后的有序度。假设数组有 n 个元素，满有序度 = n(n-1)/2

​ 因此，我们有公式：有序度 + 逆序度 = 满有序度

​ 排序过程就是增加有序度，减少逆序度，最终达到满有序度的过程。

​ 对于插入排序来说，交换会导致：有序度++， 逆序度–，因为交换的是相邻的两个元素。

​ 所以交换的次数等于逆序度，在平均情况下，逆序度 = 有序度 = n(n-1)/4。

​ 比较的次数大于等于交换的次数(先比较，再判断要不要交换)，但少于等于n(n-1)/2。

  1. 空间复杂度：O(1)
  2. 稳定性：稳定

​ 因为当两个元素相等时，我们是不会交换次序的。

1.5 希尔排序

希尔排序(Shell sort)是插入排序的一种改进版本，它是基于插入排序下面两点性质而提出的改进方法：

  • 插入排序在对几乎已经排好序的数据排序时，效率很高，可以达到线性排序的效率。

  • 但插入排序每次只能将数据移动一位，因此一般情况下比较低效。

希尔排序，又叫缩小增量排序。它通过将元素在组间交换，从而可以将小的元素快速地移动到数列的前面。前面几次组间插入排序的目的，是使数列基本有序。最后一次，增量(gap)为 1，也就是简单的插入排序了。

希尔排序的效率与 gap 序列相关，希尔本人推荐的 gap 序列为：n/2, n/4, …, 1。但这个序列并不是最佳的。

1.7 快速排序

快速排序(Quick sort)是建立在分区操作上的一种高效的排序算法。快速排序也是分治思想的一种典型应用。快速排序的步骤如下：(和严蔚敏的不一样，严蔚敏的效率低一些)。

  1. 挑选基准值：从数列中挑选一个元素作为”基准值”。

  2. 分区：将比基准值小的元素排列在数列的前端，将比基准值大的元素排列到数列的尾端，与基准值相等的元素可以放到任意一段。分区操作之后，基准值就已经就位了。

  3. 递归排序子序列：递归地将小于基准值元素的子序列和大于基准值元素的子序列排序。

​ a. 最好情况：每次分区都可以分为大小相等的两份。O(nlogn)

     `T(n) = 2T(n/2) + O(n)`

​ b. 最坏情况：每次分区，基准值最终都位于数列的两端。O(n2)

​ T(n) = T(n-1) + O(n) = T(n-1) + cn = T(n-2) + c(n-1) + cn = … = c + c*2 +… + cn = c(n+1)n/2

​ c. 一般情况：我们可以稍微极端一点，假设每次分区都分为 9:1 大小的区间。O(nlogn)

​ T(n) = T(9n/10) + T(n/10) + O(n)

​ T(n) <= cn * log10/9n

  1. 空间复杂度：O(logn)

​ 平均情况下，栈的深度为 O(logn)。

  1. 稳定性：不稳定

​ 在分区操作中，选取基准值后，我们会将其和最右边的元素进行交换。这可能是一次长距离的交换，因此快速排序是不稳定的。

改进策略

快速排序算法虽然最坏情况下的时间复杂度是 O(n2)，但是平均情况下时间复杂度是O(nlogn)。不仅如此，快速排序退化到O(n2)的概率极小，我们还通过合理地选择基准值来避免这种情况发生。

  1. 选取基准值

​ a. 指定最左边或最右边的元素为基准值。

​ b. 随机选取。

​ c. 选择 3 个或 5 个元素，然后选取其中位数作为基准值。

  1. 当元素个数比较少时(比如小于等于32个元素)，采用插入排序。
  2. 每次分区分为三个区域：<pivot，=pivot，>pivot。当相等的元素很多时，这种分区方法可以极大地提高快速排序的效率。

1.8 堆排序

堆排序(Heapsort)是指利用堆这种数据结构所设计的一种排序算法。

堆是一个近似完全二叉树的结构，并同时满足堆的性质：即子结点的 key 值总是小于或大于它的父结点的 key 值。

堆排序算法的步骤如下：

  1. 构建大顶堆

​ 找到第一个非叶子结点，从后往前构建大顶堆。

  1. 将堆顶元素和无序区的最后一个元素交换，并将无序区的长度减 1。
  2. 将无序区重新调整成大顶堆。
  3. 重复步骤2和步骤3，直到无序区的长度为 1。
